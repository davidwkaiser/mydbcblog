2. What are the best practices associated with using classes vs. ids?

So the best explanation of this question came from THIS PAGE. Below is the quote that seems to capture it best.

"Maybe a good analogy here is bar codes and serial numbers. Take an iPod in a store. On the packaging will be a bar code. This tells the store what the product is, so when it is scanned, the system knows exactly what the product is and what it costs. It might even be able to know what color it is or where it was kept in the store. All iPod of this same type have the exact same bar code on them.

"The iPod will also have a serial number on it which is absolutely unique to any other iPod (or any other device) in the world. The serial number doesn’t know the price. It could, but for the store this wouldn’t be a very efficient way to store and use that data. Much easier to use the barcode, so that for example, if the price changed, you could just change the price for that bar code and not every individual serial number in your system.

This is much like ID’s and Classes. Information that is reuseable should be kept in a class and information that is totally unique should be kept in an ID."

So bottom line, "class" is for groups of things and "IDs" are for specific individuals. I created several classes to be used in blog posts, in the course of doing this research. There is a class for "questions" and a class for "quotes," since I will likely use those items over and over again. I saw several examples that refered to individual comments on a blog post as a good use of the ID item.

3. What are the differences between relative, absolute, fixed, and static positioning?

OK this is tricky. I found a great explanation / demostration of the different types of position at THIS PAGE and a solid discussion HERE, I think I understand it, but I'm not sure how well I can explain it, but I'll try

Static is the default, it just means the element is placed where it "normally" would be assigned if you don't specify further.

Relative can be used to move an element. For example we can add "top=20px" to move the element DOWN 20 pixels relative to where "position: static" would have placed it. Although it is "moved" in terms of where we see it, other items are placed relative to it as if it were in its "static" place. Poorly explained, but I understand it...

Absolute means you can pin something to a very specific place on the page and it will stay there. However, this removes it from the flow of the items, other items will ignore it and will not position relative to it.

Fixed means that the position on the viewer stays the same, so that if you scroll down or right, for example, the fixed element remains in the same place on your screen. We may see this on those "sign up now and get a free thingie" boxes on some "squeeze pages."

4. What does it mean to display inline vs using inline block?

I found a nice visual of this at STACK OVERFLOW:

It was explained very succinctly as follows:

"An inline element has no line break before or after it, and it tolerates HTML elements next to it.

A block element has some whitespace above and below it and does not tolerate any HTML elements next to it.

An inline-block element is placed as an inline element (on the same line as adjacent content), but it behaves as a block element."

That seems to sum things up very nicely.
