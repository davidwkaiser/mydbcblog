Hashes and Arrays

Different Types of Data Structures

February 13, 2016

Hashes and arrays are both data structures in Ruby.

An ARRAY is an ordered list. Its elements can incude any data type: integers, floats, constants, variables, other arrays, and hashes. Elements are retrieved using their index, as we see in the example below.


array = [107, "text", [a,b,c]]
puts array[0]
107

Indices are assigned to each element of the array, starting with the index 0 (don't ask, most programming languages do this, there is a reason, you can read about it HERE). Anyway, the array is ordered, so in our above example, array[0] = 107, array[1] = "text," etc. The good news is that it's simple, it's basically just a list of stuff. You can repeat elements, and you can use all sorts of cool built in methods like .sort, .length, etc, to manipulate them. The downside is that you can only use numbers as indices, you can't associate abbreviations with their full names, for example. For this, in Ruby, you need a hash.

A HASH is a data structure which associates "values" with unique "keys," as we see in the example below:


months = {
"Jan" => "January",
"Feb" => "February",
"Mar" => "March",
}
puts months["Jan"]
"January"

The element on the left is the "key," it must be unique. The element on the right is the value associated with that key, it does not need to be unique. Like an array, a hash can contain all data types.

In a sense, a hash is similar to an array, except that the "key" in an array is necessarily an integer in a fixed order, while the "key" in a hash can be all sorts of things.

Like arrays, hashes have many built-in methods to manipuulate the data within them.

If you find yourself wondering whether to use an array or a hash, the diagnostic here is quite simple. If you are working with a set of items which do not require order, or can be ordered simply with integers, an array will probably do the trick. If you need to associate values with something more complex than numbers, such as abbreviations, locations, names, social security numbers, etc, then the additional complexity a hash requires will be more be worth it.
