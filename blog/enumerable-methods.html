What is an ENUMERABLE?

The best description I've seen, found at The Bastards Book of Ruby is that "The Enumerable module provides a set of methods to traverse, search, sort and manipulate collections."

An enumerable is a "mixin,"" a module imported into a class so it can be used, and the class must provide an "each" method, which yields members of the collection, one at a time.

Enumerables come with some pretty cool iterators, such as #map, #group_by and #cycle

The #map method "returns a new array filled with whatever gets returned by the block each time it runs." It's great for changing the elements in a specific way, for example, taking each one, doubling it, and putting it back.

The #group_by method creates a hash from an array or a hash, based on some criteria, such as length of the entry, alphabetical, or even random assignment.

Both very cool methods, but we are going to focus on the #cycle method for enumerables. A good explanation for #cycle is that you "use it to create an object that spits out the next item in a repeating sequence,"as we see in the snippet of code below:


# Enumerator's "next" method gives us the next day of the week
days.next
=> "Monday"

days.next
=> "Tuesday"

days.next
=> "Wednesday"

Cycle will loop indefinitely, unless we specify a number of times through the cycle. We see this in the code snippet below, from the Ruby Docs


a = ["a", "b", "c"]
a.cycle { |x| puts x }  # print, a, b, c, a, b, c,.. forever.
a.cycle(2) { |x| puts x }  # print, a, b, c, a, b, c.
Basically, it's great for generating (potentially endless) cycles. You could use #cycle to figure out the day of the week for a future date, or assign a rotating work crew over a rotating set of tasks.
