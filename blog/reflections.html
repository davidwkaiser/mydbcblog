Reflections on Web Design and Construction

The Good and The Bad

March 20, 2016

This will be short. It is basically a bit of feedback to myself on my skills around HTML, CSS and web design.

The Good:

  • I can create a basic site.
  • It works and I can maintain it properly.
  • I have written some good content.
  • When I know what to do, I am can execute it in HTML and CSS

The Bad:

  • I dislike visual design to the point of resisting it. I have a certain sense of style (I do dress very well), but I find that 1> I am not yet familiar with what distinguishes a nice web page from a crappy web page, visually, and 2> unlike clothing, which is largely toggling between options (I'm going to wear a shirt, it's just a matter of style and color), visual design of a website sems to be much more "blank slate," and I find it intimidating.
  • As a result, HTML and CSS is not as good as it could be, because I haven't practiced as much.
  • My site looks like it was built in 2002.
  • This is a problem, but one which can be surmounted.

To-Do:

  • Find people who can help me get comfortable with visual design. OR
  • Find someone who likes to design, and execute his / her vision.
  • Find practice exercises where I am creating based on specifications so I can practice HTML / CSS
  • Lighten up, make some ugly sites, then take what I have learned an use it to redesign this old 2002 site.
  • I remember from my Django course that things lilke headers and footers can be abstracted out into their own files, to ease maintenance, but I forget how to do it. I will find it and insert into these pages. THe cut and paste template is very inefficient.
