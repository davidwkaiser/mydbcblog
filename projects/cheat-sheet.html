JavaScript Syntax Cheat-Sheet

'Cuz it's hard to remember sometimes...

March 11, 2016

So, I had a lot more practice with Ruby, and Ruby is just so intuitive and pretty. As a result it took me a while get the syntax right with Javascript (by "a while," I mean "still trying to get it right"). And the JS syntax for nested data structures and objects just makes my head melt. I'm glad I'm doing this, I was complaining to a pair this morning that I have yet to find a really good explanation of it, so I might have to create it myself.

I learn and remember by working backwards from examples to principles, so that is what we will be doing here (it's my cheat sheet, I can do what I want...)

So, without further ado...

BASIC SYNTAX
FOR Loops

for (var i =1; i <= 10; i ++ ){
  console.log(i)
}

"i" is the counter, it increments by 1, stopping when equal to 10. Don't forget to include "var" to declare i, unless it has already been declared. This code will print 1 to 10.

CAUTION: use the syntax FOR X IN Y to loop over the keys of the object Y, DO NOT try to use like Ruby FOR X IN Y.

WHILE Loops

var i = 1
while (i <=10){
  console.log(i);
  i++
}

This code also counts to 10.

IF~ELSE

Here is a very basic IF statement, with an optional ELSE. These can be chained by adding ELSE IF plus a block of code. 


var x = 3
if (x % 2 === 0 ){
  console.log("it's even!")
} else {
  console.log("it's odd!")
}

If we abstract this we get: 


if(condition) {
do this block of code
}
OBJECTS

I was just starting to get Objects in Ruby and we switched to Javascript, where an "object" is a weird hybrid of a ruby hash and ruby object. On top of that, there are two different ways to access the values in JS nested data structures.

So we'll start with a tiny object, which we'll "borrow" from a recent homework assignment.


var votes = {
  "Alex": { president: "Bob", vicePresident: "Devin", secretary: "Gail", treasurer: "Kerry" },
  "Bob": { president: "Mary", vicePresident: "Hermann", secretary: "Fred", treasurer: "Ivy" },
  "Cindy": { president: "Cindy", vicePresident: "Hermann", secretary: "Bob", treasurer: "Bob" }
}

We can have all sort of fun with this object. If we want to print out just the NAMES of the NAME: VALUE pairs, we can use this code:


for (var vote in votes){
console.log(vote)
}

This will yield the following: 


Alex
Bob
Cindy

if we want just the VALUES, we can try this:


for (var vote in votes){
console.log(votes[vote])
}

This will yield:


{ president: 'Bob',
  vicePresident: 'Devin',
  secretary: 'Gail',
  treasurer: 'Kerry' }
{ president: 'Mary',
  vicePresident: 'Hermann',
  secretary: 'Fred',
  treasurer: 'Ivy' }
{ president: 'Cindy',
  vicePresident: 'Hermann',
  secretary: 'Bob',
  treasurer: 'Bob' }

Note that VOTE is in brackets, but is NOT in quotes. That is because it is a variable. Names must be in quotes, variables must not be in quotes. Dot notation can be used only for established names, so while the first expression below will yield UNDEFINED, the second and third will work. We can use dot or bracket notation because Alex is an established name.


console.log(votes.vote) => undefined

console.log(votes.Alex)
OR console.log(votes["Alex"])=>
{ president: 'Bob',
  vicePresident: 'Devin',
  secretary: 'Gail',
  treasurer: 'Kerry' }

Are we having fun yet? I'm not sure this is fun, but it is definitely helping me remember stuff, I'm already looking forward to using it.

Now let's say we want to access Alex's vote for Treasurer. We would use ANY of the expressions below:


console.log(votes.Alex.treasurer)
console.log(votes["Alex"].treasurer)
console.log(votes.Alex["treasurer"])
console.log(votes["Alex"]["treasurer"]) => Kerry

That said, just because we CAN use any of the above expressions, doesn't mean we SHOULD. It is best practice to use either dot or bracket consistently as much as possible.
